# Voice Wrapper for Claude Code — 설계 문서

**작성일:** 2026-05-22  
**상태:** 승인됨

---

## 1. 개요

개발자가 출퇴근 중 음성으로 Claude Code와 대화할 수 있는 모바일 앱 서비스. 사용자는 마치 동료 개발자와 전화 통화하듯 개발 업무를 지시하고, 작업 결과를 음성으로 보고받는다. 화면을 보지 않아도 완전히 동작하는 100% 음성 완결형 UX를 목표로 한다.

**핵심 사용 시나리오:** 출퇴근 중 차 안에서 "로그인 버그 고쳐줘" → 에이전트가 파일 수정·테스트 실행 → "수정 완료, 테스트 3개 통과했습니다" 음성 보고

## 2. 시스템 아키텍처

### 구성 요소

```
📱 Flutter 앱 (iOS/Android)
    ↕ WebSocket (인터넷)
☁️ 릴레이 서버 (Python/FastAPI, VPS)
    ↕ WebSocket (인터넷)
🖥 데스크탑 에이전트 (Python 데몬)
    ↕ subprocess (로컬)
⚡ Claude Code CLI (기존 설치, Pro/Max 구독 사용)
```

### 데이터 흐름

1. 사용자가 Push-to-Talk 버튼을 누르고 음성 명령 발화
2. Flutter 앱이 기기 내장 STT로 음성 → 텍스트 변환
3. 텍스트를 WebSocket으로 릴레이 서버에 전송
4. 릴레이 서버가 해당 사용자의 데스크탑 에이전트로 메시지 중계
5. 데스크탑 에이전트가 음성 최적화 지시어를 앞에 붙여 Claude Code CLI 호출
6. Claude Code가 실제 코딩 작업 수행 (파일 수정, 테스트 실행, 커밋 등)
7. 응답을 역순으로 중계하여 앱에 전달
8. 앱이 기기 내장 TTS로 텍스트 → 음성 재생

### Claude Code 사용 방식

- 비대화형 모드로 subprocess 호출: `claude -p "메시지"` (`-p` / `--print`)
- 대화 연속성: `-c` / `--continue` 플래그로 현재 디렉토리의 가장 최근 대화를 자동으로 이어받음
- 권한 자동 수락: `--permission-mode auto`로 파일 수정·명령 실행 시 permission 프롬프트 없이 진행
- 사용자의 기존 Pro/Max 구독 그대로 사용 — 추가 API 비용 없음
- 사용자 1명당 데스크탑 에이전트 1개 → Claude Code 세션 1개 구조

```bash
# 첫 메시지
claude -p --permission-mode auto "음성지시어\n사용자명령"

# 이후 메시지 (대화 이어받기)
claude -p -c --permission-mode auto "음성지시어\n사용자명령"
```

## 3. 컴포넌트 상세

### 3.1 Flutter 앱 (iOS / Android)

**기술 스택:** Flutter, Dart, `speech_to_text`, `flutter_tts`, `web_socket_channel`

**화면 구성 (2개):**
- 로그인 화면: 이메일 + 비밀번호
- 메인 화면: 단일 화면, 상태에 따라 표시 변경

**메인 화면 상태:**
| 상태 | 색상 | 설명 |
|------|------|------|
| 대기 | 회색 | 준비됨, 마이크 버튼 활성 |
| 녹음 중 | 초록 | Push-to-Talk 버튼 누르는 중 |
| 처리 중 | 보라 | Claude Code 작업 중, 버튼 비활성 |
| 말하는 중 | 파랑 | TTS 재생 중, 버튼 비활성 |
| 에이전트 오프라인 | 빨강 | 데스크탑 에이전트 미연결, 버튼 비활성 |

**입력 방식:** Push-to-Talk (버튼 누르는 동안 녹음, 손 떼면 전송)

**STT/TTS:**
- MVP: 기기 내장 STT/TTS (무료)
- 업그레이드 경로: OpenAI Whisper API (기술 용어 인식률 개선), ElevenLabs (자연스러운 음성)

### 3.2 릴레이 서버

**기술 스택:** Python, FastAPI, WebSockets, JWT, SQLite (사용자 정보)

**역할:**
- 사용자 회원가입/로그인 및 JWT 발급
- 앱과 데스크탑 에이전트 간 WebSocket 세션 관리
- 메시지 중계 (저장 없음, 통과만 함)
- 에이전트 온라인 여부 상태 관리

**배포:** DigitalOcean / Fly.io / Railway 등 저렴한 VPS (월 $5~10)

**비용 확장성:** STT/TTS가 기기에서 처리되므로 서버는 텍스트만 중계 → 수천 명 사용자도 $20/월 수준 유지 가능

### 3.3 데스크탑 에이전트

**기술 스택:** Python, `websockets`, `subprocess`

**역할:**
- 백그라운드 데몬으로 실행 (macOS: launchd, Windows: Task Scheduler, Linux: systemd)
- 릴레이 서버에 WebSocket 연결 유지 (재연결 로직 포함)
- 사용자 명령 수신 → 음성 최적화 지시어 추가 → Claude Code CLI 호출
- Claude Code 응답 수신 → 릴레이로 전송
- Claude Code session ID 관리로 대화 연속성 유지

**음성 최적화 프롬프트 전략:**

매 요청마다 사용자 명령 앞에 다음 지시어를 자동 삽입:

```
당신은 음성 인터페이스를 통해 개발자와 대화하는 코딩 어시스턴트입니다.
반드시 다음 규칙을 따르세요:
- 한국어로 답변
- 구어체, 전화 통화하듯 자연스럽게
- 상태 보고는 1~2문장으로
- 코드 블록, 마크다운 절대 사용 금지
- 불확실한 결정은 사용자에게 질문
- 작업 완료 시 무엇을 했는지 짧게 요약
```

## 4. 음성 UX 설계

### 대화 원칙

- 코드를 읽지 않는다: "47번째 줄 수정" 수준으로 요약
- 진행 상황을 실시간으로 보고: "파일 확인 중이에요", "테스트 실행할게요"
- 결정이 필요하면 반드시 질문: "같은 이름 파일이 있어요, 덮어쓸까요?"
- 완료 시 짧게 보고: "완료했습니다. 테스트 통과했어요."

### 대화 예시

```
사용자: "로그인 버그 고쳐줘"
에이전트: "auth.py 파일 확인하고 있어요."
에이전트: "토큰 만료 처리가 빠져있었어요. 조건문 추가했고 테스트 3개 통과했습니다."
사용자: "커밋해줘"
에이전트: "'fix: 토큰 만료 예외처리 추가'로 커밋할게요, 괜찮으세요?"
사용자: "응"
에이전트: "커밋 완료했습니다."
```

### 웨이크워드 (향후 계획)

MVP는 Push-to-Talk. 이후 Porcupine(Picovoice) 라이브러리로 웨이크워드 추가 가능. iOS 백그라운드 마이크 제한, 배터리 소모, 앱스토어 심사 이슈로 인해 선택적 옵트인 기능으로 제공.

## 5. 인증 및 보안

- 사용자 인증: 이메일 + 비밀번호, JWT 토큰
- 릴레이 서버: 모든 WebSocket 연결에 JWT 검증
- 데스크탑 에이전트: 로컬에서만 Claude Code 실행 (코드가 서버에 전송되지 않음)
- 음성 데이터: 기기에서 텍스트로 변환 후 전송 (오디오 파일 미전송)

## 7. MVP 범위

**포함:**
- Flutter 앱 (iOS/Android) — 로그인, Push-to-Talk, 상태 표시
- 릴레이 서버 — 인증, WebSocket 중계
- 데스크탑 에이전트 — Claude Code CLI 호출, 음성 프롬프트 래핑
- 기기 내장 STT/TTS

**MVP 이후:**
- 웨이크워드 (Porcupine)
- 고품질 STT (Whisper API)
- 고품질 TTS (ElevenLabs)
- 사용 기록 / 히스토리 화면
- 팀 공유 기능
